## ⚡ Быстрый старт

```bash
# 1. Форкните репозиторий на GitHub

# 2. Клонируйте свой форк
git clone https://github.com/<your-username>/AmneziaWG-Architect.git
cd AmneziaWG-Architect

# 3. Установите зависимости (bun рекомендуется, npm тоже работает)
bun install

# 4. Запустите dev-сервер
bun run dev

# 5. Откройте http://localhost:5173/ в браузере
```

### Полезные команды

| Команда | Описание |
|:---|:---|
| `bun run dev` | Dev-сервер с HMR |
| `bun run build` | Проверка типов (`vue-tsc`) + продакшн-билд |
| `bun run preview` | Превью собранного билда |

## 🗂 Структура проекта

```
src/
├── views/                 # Страницы (каждая — отдельный маршрут)
│   ├── HomeView.vue       # Генератор обфускации
│   ├── MergeKeysView.vue  # Обновление и объединение vpn://-ключей
│   ├── AboutView.vue      # О проекте
│   └── IaaView.vue        # Install AmneziaWG Anywhere (WIP)
├── components/            # Переиспользуемые компоненты
│   ├── MainHeader.vue     # Навигация
│   └── MainFooter.vue     # Подвал
├── composables/           # Vue composables (реактивная логика)
│   ├── useGenerator.ts    # Состояние и действия генератора
│   └── useMergeKeys.ts    # Состояние и действия MergeKeys
├── utils/                 # Чистые функции без зависимости от Vue
│   ├── generator.ts       # Ядро генерации: профили, CPS-сигнатуры, H/S/Jc
│   └── mergekeys.ts       # Кодеки vpn://, pako-обёртки, патчинг конфигов
├── router/                # Определение маршрутов
├── App.vue                # Корневой компонент
└── main.ts                # Точка входа
```

### Принципы архитектуры

- **`utils/`** — чистые функции, не зависят от Vue. Легко тестировать.
- **`composables/`** — реактивные обёртки поверх `utils/`. Возвращают `ref()`, `computed()`, экшены.
- **`views/`** — страницы. Содержат шаблон, стили (`<style scoped>`) и минимум логики — делегируют composables.
- **Стили** — `<style scoped>` внутри каждого компонента + глобальные CSS-переменные в `assets/main.css`.

## 🎨 Стиль кода

### TypeScript

- Строгая типизация — избегайте `any` без крайней необходимости
- Интерфейсы предпочтительнее типов для объектов
- Функции-генераторы и утилиты должны быть чистыми (без побочных эффектов)
- JSDoc-комментарии для экспортируемых функций

### Vue-компоненты

- Используйте `<script setup lang="ts">` — никаких Options API
- `<style scoped>` — стили всегда скопированы
- Реактивное состояние — `ref()` для примитивов, `reactive()` для объектов
- Именование: `PascalCase` для компонентов, `camelCase` для переменных и функций

### CSS

- Используйте CSS-переменные из `assets/main.css` (`--bg`, `--text`, `--accent`, `--amber`, и т.д.)
- Анимации через `--ease`, `--trans-fast`, `--trans-norm`
- Радиусы: `--radius-sm` (8px), `--radius` (12px), `--radius-lg` (16px), `--radius-xl` (20px)
- Адаптивность: breakpoints на `960px`, `768px`, `480px`

### Форматирование

- Отступы: **4 пробела** (настроено в Vite/TS)
- Точки с запятой: **да**
- Кавычки: **двойные** в шаблонах, без предпочтения в TS
- Trailing commas: **да** (ES5+)

## 🚀 Создание Pull Request

### Ветки

Создавайте ветку от `main`:

```bash
git checkout -b feature/my-awesome-feature
# или
git checkout -b fix/issue-42
```

**Именование веток:**

| Префикс | Когда использовать |
|:---|:---|
| `feature/` | Новая функциональность |
| `fix/` | Исправление бага |
| `refactor/` | Рефакторинг без изменения поведения |
| `docs/` | Документация |
| `style/` | Стили, анимации, UI-правки |

### Коммиты

Используйте осмысленные сообщения:

```
feat: add copy button to CPS param group
fix: history badge overflow on mobile
refactor: extract timeline data to composable
docs: update README with new architecture
style: add hover animation to merge banner
```

### Чеклист перед PR

- [ ] `bun run build` проходит без ошибок
- [ ] Протестировано в Chrome и Firefox
- [ ] Адаптивность проверена (мобильная ширина)
- [ ] Стили используют CSS-переменные (не хардкод цветов)
- [ ] Нет `console.log` в продакшн-коде
- [ ] README обновлён (если менялась структура или API)

### Описание PR

В описании укажите:
1. **Что** меняется
2. **Почему** это нужно
3. **Скриншот** (если есть визуальные изменения)

## 📦 Типы вклада

### 🐛 Исправление бага

1. Убедитесь, что баг ещё не зарепорчен в [Issues](https://github.com/Vadim-Khristenko/AmneziaWG-Architect/issues)
2. Создайте issue (или сошлитесь на существующий)
3. Исправьте и откройте PR

### ✨ Новая фича

1. Откройте issue с описанием идеи — обсудим целесообразность
2. Дождитесь одобрения (чтобы не делать работу впустую)
3. Реализуйте и откройте PR

### 🎨 UI/UX улучшения

Анимации, микровзаимодействия, адаптивность — всегда приветствуются. Просто убедитесь, что:
- Используются CSS-переменные проекта
- Нет рваных переходов при смене состояний
- Мобильная версия не сломана

### 📝 Документация

Любые улучшения README, JSDoc-комментариев, этого файла — без предварительного согласования.

## 🌐 Предложения по доменам

Пулы доменов для профилей мимикрии поддерживаются вручную и требуют верификации.

### Добавить домен

Откройте [issue](https://github.com/Vadim-Khristenko/AmneziaWG-Architect/issues/new) и укажите:

- **Домен** — например, `cdn.example.com`
- **Профиль** — `quic_initial`, `tls_client_hello`, `dtls`, `sip`, `quic_0rtt`
- **Регион** — страна и провайдер
- **Подтверждение** — вывод `curl -v`, `traceroute` или аналогичной проверки
- **Дата проверки**

### Сообщить о недоступном домене

Откройте issue с:

- **Домен** и **профиль**, в котором он находится
- **Регион** и **ISP**
- **Лог проверки** — `curl -v`, `traceroute`, скриншот
- **Дата проверки**

> ⚠️ Не добавляйте в пулы домены, которые вы не проверили лично на доступность в целевом регионе.

## 🐞 Баг-репорты

Хороший баг-репорт содержит:

1. **Что произошло** — фактическое поведение
2. **Что ожидалось** — желаемое поведение
3. **Шаги воспроизведения** — минимальная последовательность действий
4. **Окружение** — браузер, ОС, версия AWG, ширина экрана
5. **Скриншот** — если это визуальный баг
